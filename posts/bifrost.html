Bifrost: typed REST networking where the request owns the contract

A close look at mtzaquia/bifrost — a small Swift networking framework that turns an endpoint into one value, one response type, and one interception pipeline.

Part 1The networking layer that grows sideways

Most networking layers begin with one generic function: give it a path, a method, maybe a body, and ask it to decode some type. Then reality arrives — authentication, shared query items, mock responses, token refresh, logging, per-endpoint headers — and that one function becomes the most conditional code in the app.

The deeper problem is not URLSession. It is that the endpoint’s inputs, wire rules, and output type live in separate places. A call site can ask the wrong endpoint for the wrong model, and middleware must reverse-engineer an untyped request to understand what is happening.

let request = GetUser(id: userID)
let user = try await AppAPI().response(for: request)
// user is GetUser.Response — inferred from the request itself

Bifrost makes the request value the unit of networking. The request is Encodable, names its own path and method, and declares an associated Decodable response. The API supplies shared policy; interceptors handle cross-cutting behavior; the call site gets one typed async operation.

The useful constraint. You cannot call response(for: GetUser(...)) and accidentally decode [Article]. The generic return type is fixed by GetUser.Response, so the endpoint contract travels as one Swift type.

Part 2The model: API, request, response

Bifrost has three pieces, with a deliberate split between shared infrastructure and endpoint-specific data.

→

Responsethe exact Decodable result for that request

Declare the shared API

struct AppAPI: API {
  let baseURL = URL(string: "https://api.example.com/v2/")!

  func queryParameters() -> [URLQueryItem] {
    [URLQueryItem(name: "api-version", value: "3")]
  }

  var jsonDecoder: JSONDecoder = {
    let decoder = JSONDecoder()
    decoder.dateDecodingStrategy = .iso8601
    return decoder
  }()
}

API also exposes the URLSession, encoder, and both interceptor arrays. Defaults keep the small case small: .shared, ordinary JSON codecs, no global query items, and no interceptors.

Put the endpoint contract in one type

struct SearchArticles: Requestable {
  let query: String
  let page: Int
  let tags: [String]

  var path: String { "articles/search" }

  struct Response: Decodable {
    let results: [Article]
    let nextPage: Int?
  }
}

Requestable itself conforms to Encodable. That is how Bifrost gets useful defaults without a parameter dictionary: a GET request’s stored properties become query items, while POST, PUT, and PATCH requests encode into the JSON body.

MethodDefault parametersTypical override
GETAll encoded properties become query items.Override queryParameters() for names or formatting.
POSTThe complete request value becomes a JSON body.Override bodyParameters(_:) for an envelope.
PUT / PATCHThe complete request value becomes a JSON body.Add query items explicitly when an endpoint needs both.
DELETENo automatic query or body parameters.Override the appropriate function when required.

Part 3From Swift value to wire request

The build order matters. Bifrost appends the request path to baseURL, then combines query items already in that URL with API-wide items and request-local items. It creates the URLRequest, sets the method, body, and request headers, and only then hands the final value to request interceptors.

Content type is explicit. Encoding a JSON body does not automatically add Content-Type: application/json. Put it in the request’s headerFields or, more commonly, in an API-wide request interceptor.

There are two important boundaries. Request interceptors see the fully built URLRequest, not a half-resolved model. Response interceptors see raw bytes plus HTTPURLResponse, before Bifrost decides whether the status is acceptable or decodes JSON.

ResultRequest phaseResponse phase
.continueRun the next interceptor, then transport.Run the next interceptor, then validate and decode.
.return(response)Skip remaining request interceptors and transport; enter the response phase.Skip remaining response interceptors; validate and decode this response.
.restartRebuild the original request and rerun the full request/response pipeline.

Part 5Request interception: mutate or mock

A request interceptor receives an InterceptionContext<Request>. The original typed request is read-only context; the built URLRequest is mutable and authoritative.

struct AddAuthorization: RequestInterceptor {
  let token: String

  func intercept<Request>(
    _ context: inout InterceptionContext<Request>
  ) async throws -> InterceptionResult<InterceptedResponse>
    where Request: Requestable {
    context.urlRequest.setValue(
      "Bearer \(token)",
      forHTTPHeaderField: "Authorization"
    )
    return .continue
  }
}

Because the generic context still exposes the concrete request at runtime, middleware can scope itself to a marker protocol or request type. An auth interceptor can ignore public endpoints; a mock can recognize exactly one request without matching URL strings.

struct MockProfile: RequestInterceptor {
  func intercept<Request>(
    _ context: inout InterceptionContext<Request>
  ) async throws -> InterceptionResult<InterceptedResponse>
    where Request: Requestable {
    guard context.request is GetProfile else { return .continue }

    let response = HTTPURLResponse(
      url: context.urlRequest.url!, statusCode: 200,
      httpVersion: nil, headerFields: ["X-Mocked": "true"]
    )!

    return .return(InterceptedResponse(
      body: Data(#"{"name":"Mocked User"}"#.utf8),
      httpResponse: response
    ))
  }
}

The mocked response does not jump directly to decoding. It enters the normal response-interceptor phase, so analytics, normalization, and validation behave the same for real and synthetic transport.

Part 6Response interception before decoding

InterceptedResponse keeps the raw Data and mutable HTTPURLResponse together. Convenience properties expose statusCode and normalized string headers. That gives response policy enough context to inspect, rewrite, replace, or restart.

struct NormalizeAccepted: ResponseInterceptor {
  func intercept(
    _ response: inout InterceptedResponse
  ) async throws -> InterceptionResult<InterceptedResponse> {
    guard response.statusCode == 202 else { return .continue }

    response.httpResponse = HTTPURLResponse(
      url: response.httpResponse.url!,
      statusCode: 200,
      httpVersion: nil,
      headerFields: response.headerFields
    )!
    return .continue
  }
}

Status validation happens after every response interceptor. That ordering is the key to recovery: a 401 is data the response phase can act on, not an error thrown too early. If no interceptor repairs it, Bifrost throws BifrostError.unsuccessfulStatusCode before decoding.

One response shape for both paths. Network and mocked responses both become InterceptedResponse. Response middleware does not need to know whether bytes came from URLSession or a request interceptor.

Part 7Refresh a 401 and replay the original request

The most interesting use of .restart is authentication recovery. The response interceptor can make a nested refresh request, update the token store, and restart. Bifrost rebuilds the original request, which lets the request interceptor inject the fresh token on the second attempt.

struct RefreshOnUnauthorized: ResponseInterceptor {
  let auth: AuthState

  func intercept(
    _ response: inout InterceptedResponse
  ) async throws -> InterceptionResult<InterceptedResponse> {
    guard response.statusCode == 401 else { return .continue }

    let refreshed = try await auth.api.response(for: RefreshToken())
    auth.token = refreshed.token
    return .restart
  }
}

The source test for this flow observes three transports: protected with the stale token, refresh without an authorization header, then protected with the fresh token. The final response can still pass through later response interceptors before decoding.

Restart needs a stop condition. Bifrost’s pipeline uses an open while true loop and does not impose a retry limit. A response interceptor must update state or track attempts so a persistent 401 cannot restart forever.

Part 8Errors, cancellation, logging, and sharp edges

Cancellation stays native

Bifrost uses URLSession.data(for:) and checks Task.checkCancellation() immediately before and after transport. Interceptor errors and decoding errors propagate without wrapping, so callers can use ordinary structured-concurrency cancellation and targeted catch clauses.

Empty responses are explicit

struct DeleteAccount: Requestable {
  var method: HTTPMethod { .delete }
  var path: String { "account" }
  typealias Response = EmptyResponse
}

try await api.response(for: DeleteAccount())

EmptyResponse is special-cased after status validation, so a successful empty body does not go through JSONDecoder.

Logging is deliberately small

Every real transport logs the HTTP method and URL through OSLog. Set BifrostLogging.isDebugLoggingEnabled on the main actor to include bodies, headers, status, and response headers. Request-interceptor mocks bypass transport, so they also bypass transport logging.

BehaviorWhat the implementation doesWhat to decide in your app
Successful statusAccepts 200..<400.Decide whether 3xx bodies should really decode as success for your API.
JSON headersDoes not add a content type automatically.Centralize Content-Type and Accept in an interceptor.
RestartHas no built-in attempt limit.Make restart conditions stateful and finite.
GET arraysProduces repeated query names, one item per value.Override encoding when the server expects comma-separated values.

API configuration — API 
protocol API {
  var baseURL: URL { get }
  var urlSession: URLSession { get }
  func queryParameters() -> [URLQueryItem]
  var jsonEncoder: JSONEncoder { get }
  var jsonDecoder: JSONDecoder { get }
  var requestInterceptors: [any RequestInterceptor] { get }
  var responseInterceptors: [any ResponseInterceptor] { get }
}

func response<Request: Requestable>(
  for request: Request
) async throws -> Request.Response
Endpoint contract — Requestable 
protocol Requestable: Encodable {
  associatedtype Response: Decodable
  var path: String { get }
  var method: HTTPMethod { get }
  var headerFields: [String: String] { get }
  func queryParameters() throws -> [URLQueryItem]
  func bodyParameters(_ encoder: JSONEncoder) throws -> Data?
}

enum HTTPMethod { case get, post, put, patch, delete }
struct EmptyResponse: Decodable {}
Interception — context, response, and outcomes 
enum InterceptionResult<Value> {
  case `continue`
  case `return`(Value)
  case restart
}

struct InterceptionContext<Request: Requestable> {
  let request: Request
  var urlRequest: URLRequest
}

struct InterceptedResponse {
  var body: Data
  var httpResponse: HTTPURLResponse
  var statusCode: Int { get }
  var headerFields: [String: String] { get }
}

Where it shines, where to be careful

👍 Shines⚠️ Watch for
Request and response types form one compile-time endpoint contract.The whole request model is encoded by default; exclude local-only state with custom Codable or parameter overrides.
Small APIs stay small: conform two structs and call one async function.Content headers and API-specific status policy remain explicit application decisions.
Typed request context makes auth, selective mocking, and middleware easier to scope.A restart interceptor must guarantee eventual progress.
Raw response interception enables token refresh and normalization before errors or decoding.The framework is intentionally REST/JSON-shaped rather than a general transport abstraction.
A custom URLSession plus request short-circuiting keeps tests deterministic.Very large APIs still need naming and folder conventions around their many request types.

At its core, Bifrost is less about hiding URLSession than about putting the endpoint contract in the right place. Once the request owns its inputs and response type, the API can focus on shared policy and interceptors can work at precise boundaries. The result is a networking layer that remains readable at the call site without giving up the hooks real applications eventually need.
